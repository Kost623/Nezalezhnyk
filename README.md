# Nezalezhnyk

Відкритий проект з розробки процесорної архітектури (CPU), а в перспективі — GPU, RAM та SSD, повністю з відкритим кодом на всіх рівнях, без прихованих закритих підсистем.

## Філософія проекту

Сучасні комерційні процесори (Intel, AMD) містять закриті керуючі підсистеми — Intel Management Engine, AMD Platform Security Processor — які працюють нижче рівня операційної системи (так званий "ring -3"), мають повний доступ до системи і код яких не опубліковано і не піддається аудиту.

**Nezalezhnyk будується на протилежному принципі:**

- Повністю відкритий ISA (набір інструкцій) — без пропрієтарних розширень
- Відсутність будь-якого прихованого рівня привілеїв нижче ядра ОС (ring -3, -2 у закритому вигляді)
- Весь код прошивки, мікрокоду та інструментів розробки — open source
- Кожен може перевірити, що робить чип, на будь-якому рівні

## Цільові характеристики

| Параметр | Ціль |
|---|---|
| ISA | Власний відкритий набір інструкцій |
| Архітектура пам'яті | Визначається у ADR (див. `docs/architecture/`) |
| Енергоспоживання | до 40 Вт (TDP) |
| Основна платформа | Linux (першочергова оптимізація під ядро Linux) |
| Windows/macOS | Не підтримуються офіційно — відсутність портів ядра, а не штучне блокування |
| Ліцензія коду | Apache 2.0 (див. `LICENSE`) |

## Структура репозиторію

```
Nezalezhnyk/
├── src/          — вихідний код симулятора/дизайнера архітектури (C++)
├── include/      — публічні заголовкові файли
├── docs/         — документація, ADR (Architecture Decision Records), теорія
├── tests/        — модульні та інтеграційні тести
└── tools/        — допоміжні скрипти та утиліти
```

## Статус проекту

🟡 Рання стадія — визначення архітектурних рішень (ADR), теоретична база.

## Стек розробки

- Мова: C++ (компілятор GCC)
- Середовище: VSCodium + clangd + Code Runner
- ОС розробки: Arch Linux
