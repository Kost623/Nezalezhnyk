# ADR 005: Базовий набір інструкцій

Дата: 2026-07-11
Статус: Прийнято

## Контекст

ADR 002–004 визначили формати, розрядність і структуру opcode-простору. Тепер потрібно зафіксувати конкретний список інструкцій (мнемонік) базового ISA — мінімальний набір, достатній для роботи компілятора й запуску простих програм під Linux.

## Рішення

Нижче — повний базовий набір, розподілений по групах з ADR 004. У межах кожної групи `opcode[4:0]` (позначено як "під-код") відрізняє формат (R/I/S/B/J), а `funct3`/`funct7` — конкретну операцію.

### Група 0 — ALU (opcode 0–31)

**Під-код 0: R-тип, регістр-регістр**

| Мнемоніка | funct3 | funct7 | Операція |
|---|---|---|---|
| `ADD` | 000 | 0000000 | rd = rs1 + rs2 |
| `SUB` | 000 | 0100000 | rd = rs1 − rs2 |
| `SLL` | 001 | 0000000 | rd = rs1 << rs2 |
| `SLT` | 010 | 0000000 | rd = (rs1 < rs2) ? 1 : 0, знакове |
| `SLTU` | 011 | 0000000 | те саме, беззнакове |
| `XOR` | 100 | 0000000 | rd = rs1 ^ rs2 |
| `SRL` | 101 | 0000000 | rd = rs1 >> rs2, логічний зсув |
| `SRA` | 101 | 0100000 | rd = rs1 >> rs2, арифметичний зсув |
| `OR` | 110 | 0000000 | rd = rs1 \| rs2 |
| `AND` | 111 | 0000000 | rd = rs1 & rs2 |

**Під-код 1: I-тип, з константою**

| Мнемоніка | funct3 | Операція |
|---|---|---|
| `ADDI` | 000 | rd = rs1 + imm |
| `SLLI` | 001 | rd = rs1 << imm |
| `SLTI` | 010 | rd = (rs1 < imm) ? 1 : 0 |
| `SLTIU` | 011 | те саме, беззнакове |
| `XORI` | 100 | rd = rs1 ^ imm |
| `SRLI` | 101 | логічний зсув на imm |
| `SRAI` | 101 | арифметичний зсув на imm (розрізняється старшим бітом imm) |
| `ORI` | 110 | rd = rs1 \| imm |
| `ANDI` | 111 | rd = rs1 & imm |

**Під-код 2: J-формат, завантаження константи**

| Мнемоніка | Операція |
|---|---|
| `LUI` | rd = imm << 12 — завантажує старші 20 біт константи (той самий бітовий розклад, що й J-тип, але семантика інша: не перехід, а запис у регістр) |

### Група 1 — Пам'ять (opcode 32–63)

**Під-код 0: I-тип, завантаження (`load`)**

| Мнемоніка | funct3 | Операція |
|---|---|---|
| `LB` | 000 | Завантажити байт, знакове розширення |
| `LH` | 001 | Завантажити півслово (16 біт), знакове |
| `LW` | 010 | Завантажити слово (32 біти), знакове |
| `LD` | 011 | Завантажити подвійне слово (64 біти) |
| `LBU` | 100 | Байт, беззнакове розширення |
| `LHU` | 101 | Півслово, беззнакове |
| `LWU` | 110 | Слово, беззнакове |

**Під-код 1: S-тип, збереження (`store`)**

| Мнемоніка | funct3 | Операція |
|---|---|---|
| `SB` | 000 | Зберегти байт |
| `SH` | 001 | Зберегти півслово |
| `SW` | 010 | Зберегти слово |
| `SD` | 011 | Зберегти подвійне слово |

### Група 2 — Керування потоком (opcode 64–95)

**Під-код 0: B-тип, умовний перехід**

| Мнемоніка | funct3 | Умова |
|---|---|---|
| `BEQ` | 000 | rs1 == rs2 |
| `BNE` | 001 | rs1 != rs2 |
| `BLT` | 100 | rs1 < rs2, знакове |
| `BGE` | 101 | rs1 >= rs2, знакове |
| `BLTU` | 110 | rs1 < rs2, беззнакове |
| `BGEU` | 111 | rs1 >= rs2, беззнакове |

**Під-код 1: J-тип, безумовний перехід**

| Мнемоніка | Операція |
|---|---|
| `JAL` | rd = PC + 4 (адреса повернення); PC += imm |

**Під-код 2: I-тип, непрямий перехід**

| Мнемоніка | Операція |
|---|---|
| `JALR` | rd = PC + 4; PC = rs1 + imm |

**Під-код 3: системні інструкції**

| Мнемоніка | Операція |
|---|---|
| `ECALL` | Викликати системний виклик ОС (Linux syscall) — критично для взаємодії з ядром |
| `EBREAK` | Зупинка для налагоджувача (breakpoint) |

### Група 3 — Резерв (opcode 96–127)

Не визначено. Залишається під майбутні розширення (FPU, вектори) згідно з ADR 004.

## Підсумок

| Категорія | Кількість інструкцій |
|---|---|
| ALU (R-тип) | 10 |
| ALU (I-тип) | 9 |
| Завантаження константи | 1 |
| Пам'ять (load) | 7 |

| Умовні переходи | 6 |
| Безумовні переходи | 2 |
| Системні | 2 |
| **Разом** | **41** |

## Наслідки

- 41 інструкція — це вже повноцінний робочий мінімум: покриває арифметику, роботу з пам'яттю різних розмірів (байт/півслово/слово/подвійне слово), керування потоком і базову взаємодію з ОС через `ECALL`
- Наявність `ECALL` — необхідна умова для написання Linux-сумісного тулчейну (без нього програма не зможе робити системні виклики: читати файли, виводити текст тощо)
- Це достатня база, щоб почати писати симулятор/декодер на C++: кожен мнемонік однозначно визначається комбінацією `opcode` + `funct3` (+ `funct7` там, де потрібно)
